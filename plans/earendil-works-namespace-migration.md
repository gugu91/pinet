## Migration steps when unblocked

1. Confirm npm availability and exact versions:
   ```sh
   npm view @earendil-works/pi-coding-agent version
   npm view @earendil-works/pi-ai version
   npm view @earendil-works/pi-tui version
   ```
2. Create a fresh worktree from current `origin/main`.
3. Replace import specifiers and ambient module declarations:
   - `@mariozechner/pi-coding-agent` -> `@earendil-works/pi-coding-agent`
   - `@mariozechner/pi-ai` -> `@earendil-works/pi-ai`
   - `@mariozechner/pi-tui` -> `@earendil-works/pi-tui`
4. Update `browser-playwright/package.json` peer/dev dependencies to the new scope and version range chosen from npm.
5. Run `pnpm install --frozen-lockfile` first to confirm the lockfile is intentionally stale, then run a lockfile update (`pnpm install`) once versions are selected.
6. Verify no committed old-scope references remain except historical docs if intentionally retained:
   ```sh
   rg '@mariozechner/' -S . -g '!node_modules' -g '!.worktrees/**'
   ```
7. Validate:
   ```sh
   pnpm lint
   pnpm typecheck
   pnpm test
   ```
8. Open a PR that calls out the compatibility caveat: extensions migrated to `@earendil-works/*` will not work on Pi versions older than the Pi release that understands/publishes the new namespace.

## Compatibility caveat

Do not merge the actual namespace switch until the repo is ready to require the new Pi release. Migrated extensions will not run on older Pi versions that only provide/resolve `@mariozechner/*` packages.
